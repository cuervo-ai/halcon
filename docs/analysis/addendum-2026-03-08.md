# Sprint 1 Addendum — Undocumented Bugs Found During Implementation

Date: 2026-03-08

## BUG-001: check_control non-TUI stub accepts `&mut ()` but ControlReceiver is `Receiver<ClassicCancelSignal>`

**Discovered during**: PASO 1-A (build verification after ARCH-SYNC-1 fix)

**Location**:
- `crates/halcon-cli/src/repl/agent/provider_client.rs:258` — `check_control` non-TUI stub expects `&mut ()`
- `crates/halcon-cli/src/repl/agent_types.rs:14-15` — `ControlReceiver` (non-TUI) is `tokio::sync::mpsc::Receiver<ClassicCancelSignal>`

**Root Cause**: `agent_types.rs` was updated to add `ClassicCancelSignal` and change the non-TUI `ControlReceiver` type alias to `Receiver<ClassicCancelSignal>` (for PASO 1-C Ctrl-C support), but the `check_control` non-TUI stub in `provider_client.rs` was not updated to match — it still accepts `&mut ()`.

**Impact**: Build fails with `error[E0308]: mismatched types` when building without the `tui` feature (i.e., `--no-default-features`).

**Fix**: Update `check_control` (non-TUI stub) to accept `&mut tokio::sync::mpsc::Receiver<ClassicCancelSignal>` and handle `ClassicCancelSignal::Cancel` → `ControlAction::Cancel`. Also update any call sites in `convergence_phase.rs` and `post_batch.rs` that pass `ctrl_rx` to `check_control`.

**Fixed in**: PASO 1-C (see commit `feat(repl): Ctrl-C graceful cancellation — resolves GAP-CTRLC`)

## BUG-002: ARCH-SYNC-1 logic was duplicated inline in convergence_phase.rs

**Discovered during**: PASO 1-A investigation

**Location**: `crates/halcon-cli/src/repl/agent/convergence_phase.rs:534-539` (pre-fix)

**Description**: The ARCH-SYNC-1 fix (governance_rescue_active computed before adjudicate) was partially implemented by duplicating the SynthesisGate suppression condition inline (`last_convergence_ratio < 0.15 && rounds < 3`) instead of calling `synthesis_gate::evaluate()`. This created a maintenance risk: if the threshold changed in `synthesis_gate.rs`, the inline copy in `convergence_phase.rs` would silently diverge.

**Fix**: Replace inline condition with a direct call to `synthesis_gate::evaluate(GovernanceRescue, &ctx)` and derive `governance_rescue_active = !verdict.allow`.

**Fixed in**: PASO 1-A
